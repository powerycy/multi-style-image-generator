# 游戏视觉风格规范

本文档是 `multi-style-image-generator` 的内置视觉规范。写提示词或直接出图前，先选择风格，再按对应规则组装 prompt。

## 1. 通用规则

默认做“游戏实况截图 / 动画剧集镜头 / 可游玩画面”而不是海报、壁纸、电影概念图或纯插画。画面应有明确场景主体、可识别玩法时刻或叙事瞬间、少量风格元素和清晰禁用项。

直接出图 prompt 可以用英文写以提高图像模型稳定性；如果用户要求 UI 文字或中文界面，可读文字必须用中文。只写提示词时用中文结构化模板。

角色边界：

- 默认使用原创角色、原创怪物、原创伙伴或泛化职业。
- 只有用户明确点名官方角色时，才写具体官方角色。
- 商业或公开品牌用途优先泛化，不写官方游戏名、作品名、官方角色名或商标。

UI 模式：

- **全量 UI**：适合用户说“都显示、带 UI、像游戏截图、显示 HUD”。包含该游戏类型自然会出现的 UI，但不要堆满屏幕。
- **轻量 UI**：默认模式。只保留少量关键 UI，增强游戏实况感。
- **无 UI**：用户说“干净、不要 UI、不要 HUD、不要文字”时使用。写明“无 HUD、无 UI、无任何可读文字”。

视图模式：

- **普通视图**：默认模式，适合普通游戏截图、动画镜头、可游玩画面。
- **360 度环景照模式**：用户说“360 度环景照、全景图、环景照、等距柱状投影、VR 全景”等时启用。最终提示词和直接出图 prompt 必须原样加入：

```text
360 度等距柱状投影图像。
```

360 度环景照模式要求水平 360 度连续环境、2:1 宽高比（例如 4096x2048 或 2048x1024）、左右边缘可无缝衔接、主体分布在环绕空间中。不要写特写镜头、浅景深、单一正面构图、强裁切、竖构图、海报构图。

360 度交互预览：

- 当用户要求直接出图，且视图模式是 360 度环景照模式时，出图后尽量创建可交互预览，而不只回复静态图片。
- 如果可以拿到图像文件路径，直接使用该路径。
- 如果没有返回可访问路径，运行 `scripts/extract_latest_image_from_session.py --out-dir output/imagegen --name <slug>`，尝试从当前 Codex session JSONL 的 `data:image/...;base64,...` 或 image generation `payload.result` 裸 base64 数据解码出最近图片。
- 拿到图片文件后，检查是否为 2:1；如果不是，运行 `scripts/normalize_equirectangular_aspect.py <image-path>` 生成 `<stem>-2x1.png`，并使用该 2:1 文件作为最终图片。
- 运行 `scripts/create_panorama_viewer.py <2:1-image-path> --embed-image` 生成单文件本地 HTML。该预览用原生 WebGL 方向采样查看 360 效果。
- 如果用户要求“动态、动起来、动态环景、灵气粒子、云雾流动、自动巡游”等，但没有明确要求输出视频，运行 `scripts/create_dynamic_panorama_viewer.py <2:1-image-path>` 生成动态增强版 HTML。该预览仍使用静态 2:1 图作为底图，通过自动巡游、轻微 FOV 呼吸、云雾层、灵气粒子和光晕制造动态感；最终要说明它不是视频，不能让建筑或主体真实变形。
- 最终回复附上图片本地文件链接和 HTML 本地文件链接；在 Codex 桌面环境且浏览器工具可用时，可以直接打开该 HTML。
- 注意：对话里显示图片不等于 session JSONL 一定包含可提取的 `data:image`。如果既拿不到路径，也无法从 session 日志提取图片，只展示静态图并说明内置图像工具没有可落盘数据；如果用户要求保证落盘和 360 HTML，必须改走 CLI/API fallback，并按 `imagegen` 技能规则先确认该 fallback。

通用禁止项：

- 不要写实摄影、电影 CG、品牌广告、水印、二维码、乱码、伪英文、外挂字幕。
- 不要让风格元素、特效或角色盖过用户指定主体。
- 不要生成标题页、Logo 页、游戏封面，除非用户明确要求。
- 不要把真实地点改到不可识别。

## 2. 原神风格

用途：真实名胜游戏化、开放世界探索、二次元 3D 冒险、元素解谜、明亮幻想场景。

核心画风：

```text
原神开放世界游戏实况截图风格；明亮通透的二次元 3D 渲染、干净色彩、柔和环境光；不要写实摄影、不要电影 CG、不要暗黑写实、不要厚重油画质感
```

地区锚点：

- 蒙德：草地、风车、湖泊、酒庄、欧式村镇、自由冒险。
- 璃月：山水、石林、港口、亭台、商铺、琉璃瓦、岩元素古迹。
- 稻妻：海岛、神社、雷樱、木构建筑、紫色雷元素、武家城镇。
- 须弥：雨林、学院、沙漠遗迹、绿洲、蘑菇生态、草元素机关。
- 枫丹：水城、剧院、机械机关、喷泉、审判庭、优雅蓝白城市感。
- 纳塔：火山、部族聚落、热烈色彩、龙形伙伴、火元素竞技感。
- 至冬：雪原、冰宫、厚重军装、冰元素、冷峻北国城市。

元素池：

```text
原创旅行者、派蒙式小向导、传送锚点、元素方碑、七天神像式石像、宝箱、晶蝶、史莱姆、丘丘人营地、风之翼、元素反应光效、仙灵、遗迹机关
```

UI：

- 全量 UI：地点发现提示、小地图、队伍/技能图标、体力条、中文交互提示。
- 轻量 UI：少量地点发现提示和一两个 HUD 元素。
- 无 UI：无 HUD、无 UI、无任何可读文字。

真实地点 UI 写法：

```text
地点 UI 文字：顶部「探索发现」/ 中间「<真实主地名>」/ 底部「<真实上级区域名>」
```

不要把蒙德、璃月港、稻妻城、须弥城、枫丹廷等游戏内地名写进真实地点 UI。

题材锚点：

- 山林古道：雾气、松柏、石阶、残破牌坊、山神庙。
- 佛窟寺院：石窟、壁画、香炉、古钟、佛像剪影、残灯。
- 妖怪洞府：枯木、骨饰、兽皮、法器、火盆、怪石。
- 雪山荒寺：风雪、破庙、冰封石阶、苍白日光。
- 火焰山/丹炉：赤岩、灰烬、炉火、铁链、热浪。

元素池：

```text
原创行者、长棍、葫芦、破旧袈裟、妖怪剪影、山神像、古刹香炉、石窟壁画、符箓、铜铃、残破经幡、兽面门环、雾中怪影、金色法术火花
```

动作：

- 探索：原创行者沿石阶进入古寺、举火查看壁画、靠近山神像。
- 战斗：长棍架势、闪避妖怪扑击、蓄力重击、法术火花迸发。

UI：

- 全量 UI：少量血条/气力条、法术图标、葫芦次数、敌人名牌，中文界面。
- 轻量 UI：只保留血条或交互提示。
- 无 UI：无 HUD、无 UI、无任何可读文字。

避免：

- 不要写成西游动画片、可爱猴子、儿童插画、欧美恶魔城、漫威英雄。
- 不要自动写官方角色名；默认写“原创行者”或“持棍修行者”。

## 9. 直接出图 prompt 组装

当用户要求直接生成图片时，把结构化提示词压成一段适合图像生成工具的 prompt。

必须包含：

- 用户主体：地点、产品、人物、物体或场景。
- 选定视觉风格：按本文档选一个明确风格。
- 玩法时刻：角色正在探索、对战、种田、潜水、经营、调查等。
- 风格元素：自然嵌入 3-7 个，不要排成摆拍清单。
- UI 模式：全量 UI、轻量 UI 或无 UI。
- 视图模式：普通视图或 360 度环景照模式；环景照模式必须包含“360 度等距柱状投影图像。”，并明确要求 2:1 宽高比、左右边缘无缝衔接。
- 禁用项：按风格补充，不要广告、水印、二维码、乱码。

360 度环景照模式直接出图后，优先把生成图落盘；如果内置图像工具没有返回路径，尝试从 session 日志解码最近图片。落盘后必须检查图片是否为 2:1；如果不是，先运行 `scripts/normalize_equirectangular_aspect.py <image-path>` 生成 2:1 文件，再用这个 2:1 文件创建交互预览 HTML 并在最终回复给出链接。若两者都失败，不要声称已创建预览；需要保证落盘时改用 CLI/API fallback。

直接出图时不要先把 prompt 发给用户再问确认，除非用户明确要求先审提示词。
